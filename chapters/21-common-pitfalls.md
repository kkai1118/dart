# 21 补充：初学者常见坑

对应主文档章节：`21. 初学者常见坑`

## 坑 1：把 `var` 当成弱类型

`var` 只是省略类型写法，不是“随便换类型”。

```dart
var count = 1;
// count = '1'; // 不可以
```

一旦 Dart 根据初始值推断出类型，后面类型就定下来了。

## 坑 2：看到 `?`、`!` 就慌

把它们拆开理解：

- `?`：允许为空
- `!`：强制告诉编译器这里不为空

最容易出错的是：

```dart
String? name = null;
// print(name!.length); // 运行时会报错
```

记住：

- `?` 不是报错
- `!` 才是有风险的那一步

如果不确定有没有值，优先考虑：

```dart
print(name?.length);
print(name ?? 'guest');
```

## 坑 3：分不清“变量不可变”和“对象不可变”

```dart
final list = [1, 2];
list.add(3); // 合法
```

这里 `final` 限制的是：

- `list` 这个变量不能重新指向别的对象

但不一定限制：

- 这个列表内部内容不能变化

如果想表达“内容也不想随便改”，通常要再结合更合适的写法或约束。

## 坑 4：函数调用和对象创建看混

```dart
User(name: 'Tom')
```

这是构造函数调用，也是在创建对象。

对初学者来说，最容易混的地方是：

- 看到 `Something(...)`
- 不知道它是函数还是类实例化

可以先这样判断：

- 如果 `Something` 是类名，大概率是在创建对象
- 如果 `something` 是函数名，大概率是在调用函数

比如：

```dart
print('hello'); // 函数调用
User(name: 'Tom'); // 创建对象
```

## 坑 5：见到长 Flutter 代码就不敢拆

其实通常就是：

- 创建对象
- 传命名参数
- 参数里再创建子对象

## 坑 6：把 `null`、空字符串、空列表混为一谈

这三个不是一回事：

- `null`：没有值
- `''`：有值，只是空字符串
- `[]`：有值，只是空列表

比如：

```dart
String? name = null;
String title = '';
var items = [];
```

它们都“看起来像空”，但语义不同。

## 坑 7：下标从 1 开始想

Dart 的 `List` 下标从 `0` 开始。

```dart
var list = ['a', 'b', 'c'];
print(list[0]); // a
```

初学者很容易把第一个元素误写成 `list[1]`。

## 坑 8：以为可选位置参数可以跳着传

不可以。

```dart
void showUser(String name, [int age = 18, String city = 'Shanghai']) {}
```

这里调用时必须按顺序来：

```dart
showUser('Tom');
showUser('Tom', 20);
showUser('Tom', 20, 'Beijing');
```

不能直接只传后面的 `city`。

## 坑 9：忘了异步代码需要 `await`

```dart
final result = fetchName();
```

这里拿到的通常不是最终结果，而是 `Future`。

如果你真正想拿到值，往往要这样写：

```dart
final name = await fetchName();
```

## 坑 10：把报错当成“自己不适合学”

初学 Dart / Flutter 时，报错非常正常。

更有效的做法是：

- 先看报错里提到的变量或类型
- 再看是编译时报错还是运行时报错
- 然后回到对应知识点拆开理解

## 一个最实用的习惯

遇到看不懂的代码时，不要整段硬扛，拆成下面几类去看：

- 这是变量还是常量
- 这是函数还是构造函数
- 这是对象还是集合
- 这里是不是有可空值
- 这里是不是异步代码
