# 01 补充：Dart 是什么

对应主文档章节：`1. Dart 是什么`

## 为什么 Flutter 选择 Dart

- 语法上手快，前端和后端开发者都比较容易迁移
- 支持 AOT 与 JIT，既方便开发调试，也兼顾发布性能
- 语言本身为 UI 场景做了不少优化，比如命名参数、集合内 `if/for`、空安全

这里的 AOT 和 JIT 可以先这样理解：

- `AOT`：提前编译，代码在发布前先编译好，适合正式上线版本
- `JIT`：即时编译，代码在运行时再处理，适合开发调试

简单记：

- 开发时更看重“改得快、调得快” -> `JIT`
- 发布时更看重“启动快、性能稳” -> `AOT`

## 你在 Flutter 里最常见到的 Dart 能力

- 用类描述页面、组件、模型对象
- 用函数组织事件处理和业务逻辑
- 用 `Future` 处理网络请求、磁盘读取等异步流程
- 用 `const` 和不可变思路提升代码稳定性

## 学 Dart 时的正确预期

- 先会读 Flutter 代码，再追求语法细节全覆盖
- 前期不用背所有语法，优先掌握高频写法
- 看到不熟悉的写法时，先拆成“变量 / 函数 / 类 / 调用”四层看

## 和 JavaScript / TypeScript 的直观差异

- Dart 的类型系统默认更严格，空安全约束更强
- Dart 没有原型链式对象模型，类和构造函数更直接
- 命名参数是 Dart 的高频特性，在 Flutter 中几乎随处可见
