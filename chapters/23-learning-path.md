# 23 补充：学习路线

对应主文档章节：`23. 学习路线`

## 为什么要按顺序学

Dart 和 Flutter 的知识点强相关，但不适合一上来全铺开。

推荐顺序的核心原因：

- 变量和类型决定你能不能读懂基础语句
- 函数和参数决定你能不能看懂 API 调用
- 类和构造函数决定你能不能看懂 Widget 树
- 异步决定你能不能读懂网络和初始化流程

## 推荐学习顺序

### 第 1 阶段：基础语法

先学这些：

- 变量
- 基础类型
- 字符串
- 条件判断
- 循环

目标是：先能看懂最基础的语句。

### 第 2 阶段：函数和参数

重点学：

- 函数定义
- 返回值
- 位置参数
- 命名参数

目标是：看懂 API 调用和 Flutter 构造函数。

### 第 3 阶段：类和对象

重点学：

- 类
- 字段
- 方法
- 构造函数
- getter

目标是：看懂对象创建和 Widget 实例化。

### 第 4 阶段：空安全和集合

重点学：

- `String?`
- `?.`
- `??`
- `List`
- `Map`
- `Set`

目标是：能处理真实业务数据，不容易被空值和集合绕晕。

### 第 5 阶段：异步、异常、import

重点学：

- `Future`
- `async`
- `await`
- `try/catch`
- `import`

目标是：能读懂网络请求、初始化和模块拆分。

## 一种更实操的学习方式

每学完一章就做一件事：

- 写 3 行示例代码
- 去项目里找一个真实例子
- 用自己的话复述一遍

这样比纯记笔记有效得多。

## 一个 7 天入门节奏

### 第 1 天

- 变量、基础类型、字符串

### 第 2 天

- `if`、`for`、`List`、`Map`

### 第 3 天

- 函数、参数

### 第 4 天

- 类、对象、构造函数、getter

### 第 5 天

- 空安全

### 第 6 天

- 异步、异常处理

### 第 7 天

- 回到 Flutter 项目里，按文件顺序读代码

## 每学完一部分怎么判断自己会了

可以问自己这几个问题：

- 我能不能自己写一个小例子
- 我能不能解释这段代码每一行在干什么
- 我能不能把这个知识点放进 Flutter 场景里理解

如果这三点都能做到，通常就是真的会了。
